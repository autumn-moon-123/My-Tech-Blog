1.freeRtos调度器的调度机制

freertos有两种调度机制，一种是抢占式调度，一种是协作式调度。
抢占式调度：高优先级任务可以直接抢占低优先级任务。
协作式调度：任务除非自己结束，无法被抢占。
这两个模式通过设置宏taskYIELD_IF_USING_PREEMPTION来选择
# if(configUSE_PREEMPTION == 0)# define taskYIELD_IF_USING_PREEMPTION()  // 空实现# else# define taskYIELD_IF_USING_PREEMPTION() portYIELD_WITHIN_API()# endif
调度模式configUSE_PREEMPTION=0
 (协作式)
configUSE_PREEMPTION=1
 (抢占式)
宏展开结果
空宏（无代码生成）
展开为 portYIELD_WITHIN_API()
触发任务切换的时机
必须显式调用 taskYIELD()
高优先级任务就绪时自动触发
典型调用场景
在 xTaskResumeAll() 中检测到待处理任务时
中断唤醒高优先级任务时

这篇文章主要会从更为常用的抢占式调度来说明。

2.freeRtos调度器的优先级机制

固定优先级
每个任务创建时分配一个优先级（0 最低，configMAX_PRIORITIES 最高）。
优先级反转处理
可通过互斥量的优先级继承（Priority Inheritance）缓解。

3.freeRtos调度器的功能

通过调度器，任务在以下状态间切换：（具体函数实现会在下篇文章分析）

就绪（Ready）
准备运行，等待调度器分配 CPU。

运行（Running）
当前正在执行的任务。

阻塞（Blocked）
等待事件（如延时、信号量、队列等）。

挂起（Suspended）
显式暂停（vTaskSuspend()），不参与调度。

4.freeRtos调度的实现原理
  1. 保存当前任务上下文（寄存器、栈指针）到其任务控制块（TCB）。
  2. 从就绪队列中选择最高优先级任务。
  3. 恢复新任务的上下文并执行。

图（调度器的工作流程）

好消息是，保存当前任务上下文和恢复新任务的上下文并执行的核心功能可以通过一个中断Pendsv来完成。
__asm void xPortPendSVHandler( void ){    extern uxCriticalNesting;    extern pxCurrentTCB;    extern vTaskSwitchContext;/* *INDENT-OFF* */    PRESERVE8    mrs r0, psp    isb    ldr r3, =pxCurrentTCB /* Get the location of the current TCB. */    ldr r2, [ r3 ]    stmdb r0 !, { r4 - r11 } /* Save the remaining registers. */    str r0, [ r2 ] /* Save the new top of stack into the first member of the TCB. */    stmdb sp !, { r3, r14 }    mov r0, #configMAX_SYSCALL_INTERRUPT_PRIORITY    msr basepri, r0    dsb    isb    bl vTaskSwitchContext    mov r0, #0    msr basepri, r0    ldmia sp !, { r3, r14 }    ldr r1, [ r3 ]    ldr r0, [ r1 ] /* The first item in pxCurrentTCB is the task top of stack. */    ldmia r0 !, { r4 - r11 } /* Pop the registers and the critical nesting count. */    msr psp, r0    isb    bx r14    nop/* *INDENT-ON* */}


pengsv是一个系统中断，上面的代码是他的中断服务函数
解释一下这个函数：
先关联到pxCurrentTCB这个全局指针变量
再通过CurrentTcb读到current任务栈的位置
然后把寄存器的值保存到栈
接着调用vTaskSwitchContext得到newTcb
之后通过newTcb得到new任务栈
最后把new任务栈保存的东西恢复到寄存器并退出

总体来说，这个中断函数保存了现在的task，启动了新的task。
而从就绪队列中选择最高优先级任务则是通过vTaskSwitchContext来完成的

这个函数是调度器选择下一个task的核心函数

如果调度器不被挂起且栈未溢出，则会调用taskSELECT_HIGHEST_PRIORITY_TASK()来找到优先级最高，就绪时间最早的任务来进行。

pendsv会调用TaskSwitchContext来确定切换task的目标。

5.freeRtos调度器的启动条件
任务主动让出 CPU
调用 taskYIELD() 或进入阻塞状态（如 vTaskDelay()）。
中断服务例程（ISR）
发送信号量、队列等后调用 portYIELD_FROM_ISR()。

系统节拍（Tick）中断
检查时间片轮转或延迟到期。

总之，任务有直接切换和延迟切换两种切换方式，它们启动任务调度器的机制不太一样。
一、即时调度
即时调度可以是任务调用taskYIELD()来直接触发pendsv（调度），或者task的一些处理函数调用taskYIELD()来触发pendsv。
举两个例子
               if( xYieldRequired != pdFALSE )                {                    taskYIELD_IF_USING_PREEMPTION();                }                else                {                    mtCOVERAGE_TEST_MARKER();                }

这一段代码出自 vTaskPrioritySet() ，该函数会修改任务的优先级，若优先级高于正在执行的task，则会调用taskYIELD_IF_USING_PREEMPTION()直接调度。


    if( xSchedulerRunning != pdFALSE )    {        /* If the created task is of a higher priority than the current task         * then it should run now. */        if( pxCurrentTCB->uxPriority < pxNewTCB->uxPriority )        {            taskYIELD_IF_USING_PREEMPTION();        }        else        {            mtCOVERAGE_TEST_MARKER();        }    }

这一段代码出自prvAddNewTaskToReadyList（），若新的ready任务优先级高于current任务，则调用taskYIELD_IF_USING_PREEMPTION()直接调度。

这两个例子说明了在抢占式调度时，优先级更高的任务就绪时，会直接调用taskYIELD_IF_USING_PREEMPTION()进行调度。
二、延迟调度：
延迟调度也是调用pendsv进行调度，但是因为中断关闭或是有其他中断正在执行尤其是定时器中断导致调度会延迟到某个时候再进行。
延迟调度的核心中断：systemtick

6.调度器的挂起机制

调度器也可以开启和关闭。
当多个任务或中断需要访问共享资源（如全局变量、队列、外设寄存器）时，若未加保护，可能导致 数据竞争（Race Condition）。传统解决方案是 关中断（taskENTER_CRITICAL()），但会阻塞所有中断响应，破坏实时性。

TaskSuspendAll(）和xTaskResumeAll()用于挂起/恢复调度器（可嵌套），这两个任务不会被中断打断，也不会禁用中断。它们会控制宏uxSchedulerSuspended的值告诉调度器是否启动。

调度器挂起后，若PendSV 被触发（如调用 taskYIELD()），调度器会检查 uxSchedulerSuspended的值：若 uxSchedulerSuspended > 0，则 跳过上下文切换，仅标记 xYieldPending = pdTRUE（延迟处理）。

恢复调度器后（uxSchedulerSuspended == 0），再处理挂起的切换请求。
